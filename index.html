대시보드 수정 (명단 관리 탭)

  • 대상자 등록 시 고유코드(예: A7K2X9) 자동 발급, 눌러서 복사 가능
  • 연동상태 표시: ⚪ 미연동 → ⏳ 승인대기 → ✅ 연동완료
  • "코드 재발급" 버튼
  • 교재 상세에 🔔 봇 알림 재발송 버튼도 추가 (그룹 수정 후 다시 알림 보낼 때 사용)

봇 동작

  1. 대상자: t.me/봇아이디 → /start → 고유코드 입력
  2. 관리자 텔레그램으로 승인 요청 도착 → 버튼으로 승인/거절
  3. 승인되면: 새 교재 등록 시 자동 알림(버튼으로 바로 확인) + 언제든 /확인 명령으로 직접 조회 가능
  4. 봇에서 확인해도, 대시보드에서 확인해도 같은 데이터라 실시간으로 양쪽 다 반영됩니다

배포 순서 (README.md에 상세히 있어요)

  1. BotFather에서 봇 생성 → 토큰 발급
  2. Firebase 콘솔 → sejong-dasnboard → 서비스 계정 → 비공개 키 생성(JSON)
  3. GitHub 새 저장소에 index.js, package.json 업로드
  4. Render.com → New Web Service → 저장소 연결 → 환경변수 3개 등록
  5. UptimeRobot으로 5분마다 핑 (무료 플랜 잠자기 방지)
  6. 배포 후 관리자가 봇에게 /관리자등록 admin1000!! 한 번 입력

진행하시다 막히는 단계(특히 서비스 계정 JSON 발급이나 Render 환경변수 등록) 있으면 스크린샷 보내주세요, 바로 짚어드릴게요.

두 가지 운영 방식을 말씀드릴게요.

① 하나로 통합
지금 만든 세종취합봇 하나에 기능을 계속 추가하는 방식이에요. 사람들이 고유코드로 딱 한 번만 연동해두면, 나중에 참석확인이나 다른 취합 기능이 생겨도 같은 계정으로 다 처리돼요. /확인, /참석 이런 식으로 명령어만 늘어나는 구조입니다.

  • 장점: 사용자 입장에서 편함 (봇 하나, 연동 한 번)
  • 단점: 기능이 많아지면 코드가 하나로 커짐 (관리는 제가 하는 부분이라 크게 부담은 없어요)

② 기능별로 분리
지금처럼 "교재열람확인봇"을 그대로 두고, 참석확인이 필요해지면 새 봇을 하나 더 만드는 방식이에요.

  • 장점: 각 기능이 독립적이라 문제 생겨도 서로 영향 없음
  • 단점: 사람들이 봇마다 따로 링크 열고 따로 연동(코드 입력)해야 함 — 번거로움

말씀하신 "세종취합봇"이라는 이름 자체가 이미 **①번(통합)**을 염두에 두신 것 같아서, 저도 ①번을 추천드려요. 어느 쪽으로 할지 골라주시면 그에 맞게 구조를 잡아드릴게요.

세종취합봇

교재열람확인, (추후) 참석확인 등 여러 "취합" 기능을 하나의 봇, 하나의 연동으로 처리하는 텔레그램 봇입니다. 회원명단·그룹·텔레그램 연동 상태는 모든 기능이 공유하므로, 사람들은 딱 한 번만 고유코드로 연동하면 됩니다.

하는 일

  • 대상자가 고유코드를 입력 → 관리자에게 승인 요청 (텔레그램 버튼으로 승인/거절)
  • 새 항목(교재/참석확인 등) 등록 시 연동된 대상자 전원에게 자동 알림 + 버튼으로 즉시 확인
  • /확인 명령으로 언제든 본인이 직접 모든 모듈의 미확인 항목을 한 번에 조회

새 기능(모듈) 추가하는 법

index.js 상단의 MODULES 배열에 정의를 하나 추가하면 끝입니다. 예를 들어 참석확인을 추가할 때:

js
{
  key: 'attendance',      // Firebase 경로: sejong_unified/attendance
  path: 'attendance',
  icon: '🙋',
  noun: '참석확인',
  detail: (d) => `일시: ${d.date || '-'}`
}

해당 대시보드가 sejong_unified/attendance/<id>에 아래 형태로 저장하도록만 맞추면, /확인과 자동알림이 자동으로 확장됩니다.

{ title, date, groupIds:{gid:true}, confirms:{}, createdAt, notified:false }

배포 순서 (Render.com)

1) 텔레그램 봇 만들기 (또는 기존 봇 이름 변경)

  1. 텔레그램에서 @BotFather 검색 → 대화 시작
  2. 새로 만드는 경우: /newbot → 이름 세종취합봇, 아이디(예: sejong_collect_bot) 입력 기존 봇이 있다면: /setname → 봇 선택 → 세종취합봇 으로 변경
  3. /setdescription → "세종기획 교재열람·참석확인 등 각종 확인을 취합하는 봇" 같은 소개문 등록
  4. 토큰(숫자:영문 조합 긴 문자열)을 복사해두기

2) Firebase 서비스 계정 키 발급

  1. Firebase 콘솔 → sejong-dasnboard 프로젝트
  2. ⚙️ 프로젝트 설정 → 서비스 계정 탭
  3. 새 비공개 키 생성 클릭 → JSON 파일 다운로드
  4. 그 파일을 텍스트 편집기로 열어서 전체 내용을 복사 (한 줄로 붙여넣기용)

3) GitHub에 업로드

  1. 새 저장소 생성 (예: sejong-unified-bot)
  2. package.json, index.js 파일 업로드 ⚠️ .env 파일은 절대 업로드하지 마세요 (비밀키 노출 위험)

4) Render.com 배포

  1. Render 대시보드 → New + → Web Service
  2. 방금 만든 GitHub 저장소 선택
  3. 설정:
    • Build Command: npm install
    • Start Command: npm start
  4. Environment 탭에서 환경변수 3개 등록:
    • TELEGRAM_BOT_TOKEN = 1단계에서 받은 토큰
    • FIREBASE_DATABASE_URL = https://sejong-dasnboard-default-rtdb.firebaseio.com
    • FIREBASE_SERVICE_ACCOUNT = 2단계 JSON 파일 전체 내용 (한 줄로)
  5. Create Web Service → 배포 완료까지 2~3분 대기

5) 잠들지 않게 설정 (무료 플랜 필수)

Render 무료 플랜은 15분간 요청이 없으면 잠듭니다. 봇이 항상 응답하려면:

  1. UptimeRobot 무료 가입
  2. 새 모니터 추가 → Render가 준 주소(https://xxxx.onrender.com) 등록
  3. 5분 간격으로 자동 접속하도록 설정 → 봇이 항상 깨어있음

6) 관리자 등록 (최초 1회, 관리자 본인이)

텔레그램에서 봇과 대화 시작 → 아래처럼 입력:

/관리자등록 admin1000!!

(대시보드 관리자 암호와 동일 — 암호를 바꿨다면 바뀐 암호로 입력)

이제 대상자들이 코드를 입력하면 이 채팅으로 승인 요청이 옵니다.

대상자 사용법 (공유용 안내문)

  1. 텔레그램에서 봇 링크 열기: t.me/봇아이디
  2. /start 입력
  3. 관리자에게 받은 고유코드 입력 (예: A7K2X9)
  4. 관리자 승인 후 자동으로 알림을 받고, /확인으로 언제든 조회 가능
